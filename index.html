算法入门

算法的五大特征：

  • (1) 输入性： 有零个或多个外部变量作为算法的输入
  • (2) 输出行：算法至少有一个量作为输出
  • (3) 确定性：算法中每条指令清晰，无歧义
  • (4) 有穷性：算办法中每条指令的执行次数有限，执行每条指令时间有限
  • (5) 可行性：算法原则上能够精确的运行，而且有限次的运算后即可完成

算法效率衡量

  • 时间复杂度

​ 一般来说，一个算法执行所消耗的时间从理论上是算不出来的，只有上机运行才能测试出来，而在不同配置的机器上所运行的时间又有所差别，但是我们知道一个算法花费的时间与算法中语句的执行次数是成正比的，这样我们就可以根据算法的执行次数得出其时间复杂度。

​ 算法的时间复杂度其实是一个函数，该函数计算的是执行基本操作的次数。一个算法语句总的执行次数是关于问题规模N的某个函数，记为f(N)，N成为问题的规模；语句总的执行次数，记为T[N],当N不断变化时，算法的执行次数的增长速率和f(N)的增长速率相同。则T[N]=O(f(N)),称O(f(N))为时间复杂度的O渐进表示法

基本计算规则

  • (1) 基本操作，即只有常数项，认为其时间复杂度为O(1)
  • (2) 顺序结构，时间复杂度按加法进行计算
  • (3) 循环结构，时间复杂度按乘法进行计算
  • (4) 分支结构，时间复杂度取最大值
  • (5) 判断一个算法的效率时，旺旺只需要关注操作数量的最高次项，其它次要项和常数项可以忽略
  • (6) 在没有特殊说明时，我们所分析的算法的时间复杂度都是值最坏时间复杂度

常见时间复杂度之间的比较

  • O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(n^3) < O(2^n) < O(n!) < O(n^n)

  • 空间复杂度

    ​ 一个程序的空间复杂度是指运行完一个程序所需内存的大小。一个程序执行时除了需要存储空间和存储本身所使用的指令、常数、变量和输入数据外，还需要一些对数据进行的工作单元和存储一些为实现计算所需信息的辅助空间。一个算法所需的存储空间用f(n)表示。S(n)=O(f(n)),其中n为问题的规模，S(n)表示空间复杂度。

具体以什么作为算法的标准，还是要以实际情况进行结合探讨，如果对时间要求较高可以将时间复杂度作为衡量标准；如果对空间要求较高可以将空间复杂度作为衡量标准
